Loader unico in supabase/functions/_shared/operativePromptsLoader.ts.

Funzioni che lo usano (scope):
- generate-email (email)
- generate-outreach (outreach + channel)
- improve-email (email-quality)
- classify-email-response (lead-status / classification)
- classify-inbound-message (classification + channel)
- agent-loop (agent-loop)
- agent-execute via contextInjection.buildContextBlock (agent-loop)
- ai-assistant (general/scope dinamico)

Bug fix 2026-04-28: contextInjection.ts riferiva `opPrompts` non definito (Director View) → sostituito con query dedicata `directorPrompts` + uso di `OperativePromptRow` type.

Funzioni esplicitamente NON loader-aware (parser tecnici, no business rules):
- whatsapp-ai-extract, linkedin-ai-extract (DOM/CSS parsers)
- parse-business-card, parse-profile-ai, categorize-content
- analyze-import-structure, process-ai-import, analyze-partner
- enrich-partner-website, batch-enrichment-worker
- ai-match-business-cards, suggest-email-groups
- ai-query-planner, harmonize-proposal-chat, sherlock-extract
- agentic-decide (no user context)
- reply-classifier (legacy, no user_id; sostituito da classify-inbound-message)
