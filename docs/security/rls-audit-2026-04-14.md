## CRITICAL Issues (Fixed in this PR)

### C1: `import-files` storage bucket was PUBLIC
- **Risk**: PII exposure (emails, phones, addresses from CSV imports)
- **Fix**: Set `public = false`, added user-scoped SELECT/INSERT policies
- **Status**: ✅ FIXED

### C2: `chat-attachments` storage bucket was PUBLIC
- **Risk**: Private conversation files accessible without auth
- **Fix**: Set `public = false`, replaced broad SELECT policy with user-scoped policies
- **Status**: ✅ FIXED

### C3: `dangerouslySetInnerHTML` without sanitization in AgentDetailDrawer
- **Risk**: XSS via agent signature HTML from DB
- **Fix**: Added `sanitizeHtml()` wrapper
- **Status**: ✅ FIXED

### C4: RA module tables had `USING(true)` policies
- **Risk**: Any authenticated user could read/modify all RA data
- **Fix**: Tables don't exist in live DB (migration `20260328` was never applied). Documented for when they're created.
- **Status**: ⚠️ DEFERRED (tables not deployed)

## MEDIUM Issues (Documented)

### M1: `team_members` — no user_id scoping
- Policies use `auth.uid() IS NOT NULL` only
- Any authenticated user can CRUD all team members
- **Recommendation**: Add ownership column or restrict writes to admin

### M2: Realtime channels — no authorization
- 10 tables published to Realtime without topic-level RLS
- Any authenticated user can subscribe to any channel
- **Recommendation**: Add RLS on `realtime.messages` (requires Supabase config)

### M3: `operators` table — encrypted credentials visible to admins
- `imap_password_encrypted` / `smtp_password_encrypted` readable by any admin
- **Recommendation**: Separate credentials table with `user_id = auth.uid()` only

### M4: `kb_entries` — public role can read system doctrine
- SELECT policy allows `public` (anon) role for `user_id IS NULL` entries
- **Recommendation**: Change target role to `authenticated`

### M5: `email_templates` — anon can read/delete ownerless templates
- Policy targets `public` role with `user_id IS NULL` branch
- **Recommendation**: Restrict to `authenticated` role

### M6: `blacklist_entries` — no per-user scoping
- Any authenticated user can read/insert/update all entries
- **Recommendation**: Add ownership or admin-only write access

### M7: `partner_certifications` — no ownership check on writes
- INSERT/UPDATE/DELETE only check `auth.uid() IS NOT NULL`
- **Recommendation**: Add ownership via partner relationship

## LOW Issues

### L2: `commercial_playbooks/workflows` — template user_id visible
